Explanations

Task

Please set weights for the different categories of PPML characteristics to reflect how valueable the fulfillment of this category would be for your use case. This means that a high weight assigned signalizes that this category of the respective characteristic is assessed by the developers as valuable, while a low weight signalizes little value for the use case. Keep in mind the exclusiveness constraint explained below when setting the weights.


Exclusiveness: If the categories for a PPML characteristic are exclusive, this means that only one category can be applicable for a certain PPML technique. Thus, the weight of each category can be selected freely in the range [0,1]. However, if the categories are not exclusive, this means that multiple categories can be applicable for one PPML technique. To prevent that the score for this characteristics exceeds 1, the sum of all non-exclusive categories has to be 1.

Location of Raw Data Storage

The location where the raw user data is stored.

Local: stored on the user's device

Distributed: stored across multiple devices or locations, often over a network (such as the internet)

Central: stored on a central server within the AI service chain

Data Size

Data size is defined by the number of rows and columns of a dataset. PPML techniques require different sizes of data to achieve a high accuracy. This characteristic indicates whether pre-trained models have to be used. The threshold is a minimum of 1,000 samples to classify as pre-trained.

Below 1,000 samples: model is not considered pre-trained

Above or equal to 1,000 samples: model is considered pre-trained as the data size exceeds the threshold

Data Quality

Data quality is influenced by outliers and missing data that cannot be handled well by all algorithms. Further, data quality is influenced by the process of removing, aggregating, obfuscating or changing data for de-identification. There is thus an inverse relation with the characteristic 'Aggregation'.

Pre-processed: data was processed, e.g. through removing, aggregating, obfuscating or changing the data for de-identification.

Raw: data was not processed

Aggregation

Aggregating data is representing statistically the attributes of several data entitis and is only useful in specific use cases. Aggregation has an inverse relation with 'Data Quality' and 'Accuracy'.

Yes: aggregation happened on raw data during de-identification

No: aggregation did not happen on raw data during de-identification

Sensitive Attributes

Plain sensitive data is one of the most important issues in PPML. To prevent the revealing of plain sensitive data, one should differentiate between several fundamental methods.

Pseudonymised: sensitive attributes replace with pseudonyms, reducing the risk of direct identification while allowing some level of data utility

Obfuscated: sensitive attributes are intentionally altered or masked to prevent clear identification, protecting the information from being easily understood

Encrypted: sensitive attributes are transformed using cryptographic algorithms into unreadble formats, ensuring that only authorized parties with the decryption key can access the original data

Removed: sensitive attributes entirely deleted from the dataset, best method to break re-identification attacks, but rarely doable if accuracy needs to be high

Heavily Processed: data processed in a way that the original data is no longer recognisable

Explainability

Explainable results help to employ proper PPML techniques.

Ante-hoc methods: methods are explainable by default (such as decision trees)

Post-hoc methods: methods applied after the model was built

Global Explainability: explain the overall general behavior of the model

Local Explainability: aim to explain individual, specific decisions

Location of Computation

For simplicity reasons, only three main locations of computation are distinguished. Computation refers to the training of the model.

Local: computation on local user devices

Distributed: computation is spread across multiple devices or nodes in a network

Central: computation on a central server or data center, e.g. cloud computation

Training Method

Not all PPML techniques are suitable for each training method. It has to be accessed whether the selected PPML techniques are suitable for the respective use case based on the training method used.

Supervised: training method involves using labeled data where the algorithm lears from input-output pairs to make accurate predictions on new data

Semi-supervised: training method combines a small amount of labeled data with a larger amount of unlabeled data to improve learnig efficiency and accuracy

Unsupervised: training method uses only unlabeled data, focusing on identifying patterns and structures within data without predefined labels

Reinforcement: training method involves an agent learning to make decisions by receiving rewards or penalties based on its actions within an environment to maximize cumulative rewards

Accuracy (F1-Score)

An accuracy threshold for the suitable accuracy metric is defined. If a model in combination with the PPML technique is not able to achieve the threshold, the PPML should not be considered further. The F1-score was used to categorize models.

< 0.84: F1-score below 0.84

0.85 - 0.89: F1-score between 0.85 and 0.89

0.9 - 0.94: F1-score between 0.9 and 0.94

0.95 - 1: F1-score between 0.95 and 1, with 1 being the maximum score achieveable

Training Time

Training time influences the adaptability of a PPML model and depends on the computational complexity of the training task. It is very hard to compare between different approachs because the complexity of models, data sets, computational resources and parties involved differ.

>24h: training time above 24 hours required

12 - 24h: trainig time between 12 and 24 hours

<12h: training time below 12 hours

Performance

The overall runtime of an application, i.e. the time between user request and response.

Attention lost: >10s: reaction time above 10s, attention is no longer kept as threshold is exceeded

Attention kept: 1.0 - 10s: reaction time between 1.0s and 10s, with the latter being the threshold of keeping the attention on the dialogue between application and user

Uninterrupted: 0.1 - 1.0s: reaction time between 0.1 and 1.0s, with the latter being the threshold where a delay is noticed but the flow of the user's thoughts is uninterrupted

Instant reaction: <0.1s reaction time is below 0.1s, this is perceived as an instant reaction of the system

Resilience against Attacks

For this characteristic, the perspective of "globally private" is used, referring to attackers external to the AI service chain. Only attacks which leak information to an exteral adversary are considered under this characteristic. It reflects whether the considered PPML approach is resilient against the respective type of attack.

Model inversion: attack where adversary uses access to a trained model to infere sensitive input data by working backwards from the model's output

Membership inference: attack where the adversary determines whether a specific data point was part of the training dataset used to build the model by analyzing the model's responses

Purpose and Access Limitation

For this characteristic, the perspective of "locally private" is considered, refering to attackers inside the AI service chain. Each member of the AI service chain needs some access to data in order to fulfill their task. For this characteristic, it is therefore evaluated whether the PPML method implements countermeasures to protect each of the respective data layers to ensure members of the AI service chain do not use the data for other purposes than intended.

Privacy of Data Entity: information about data entities (user providing data to train ML model) is protected from unauthorized access and disclosure

Privacy of User: personal activities and information of user (person using the application) is protected

Privacy of Model: intellectual property and proprietary details of the ML model itself are protected

Privacy of Result: outcomes or predictions generated by the model are protected

Technical Robustness

Technical Robustness should ensure that even with potential changes in the operating environment or the context of the model the PPML behaves reliable during the whole AI life cycle. It means that models are reliable and robust against changes and minimises unexpected and unintentional behavior. The focus lies on the susceptibility to adversial attacks, namely model poisoning.

In model poisoning attacks, the adversary tries to change the model's prediction by polluting the model with manipulated input data. The attack can be targeted to achieve a certain classification result or untargeted, just polluting the model and lowering its accuracy

Poisoning Countermeasures: countermeasures against model poisoning attacks exist

No Countermeasures: no countermeasures against model poisoning attacks are implemented

Privacy Concerns

The security of personal data is important when using the PPML technique. Nobody should continue to use the data without consent, falsify the data or grant unauthorized persons access to the data.

Automated Decision Making: Concern that with automated decision-making through algorithms, the process becomes out of control and humans are treated as numbers rather than individuals.

Unauthorized Secondary Use: Concern of a misuse of data initially collected for a certain purpose for a different purpose without consent/authorization.

Data Bias: Concern that unrepresentative data might lead to false results.

Unauthorized Access: Concern that unauthorized people can access personal data.

User Experience

The user experience plays an important role in the usage of PPML techniques.

Ease of Use: The technology is easy to use.

Adaptability: The technology can adapt to diverging circumstances, such as changes in location or daytime.

Availability: The technology can be used anytime when needed.

Performance: The technology's use will benefit me for my tasks.

Perceived Data Processing

It is important how data is processed, namely answering the questions about what the data is used for, how it is collected, whether the data is stored correctly and for which purpose.

Collection: Concern that amount of personal data collected and stored is excessive.

Data Purpose: Concern that the purpose of the data collection of personal data is not clear, legitimate and precise.

Storage Location: The geographical location of data storage and the architecture of storing the data is concerning. The transfer of data across geographical borders might reveal personal data.

Correctness of Stored Data: Concern that the stored data is flawed or contains wrong information.

Preceived Trustworthiness

The technology used has to be trustworthy. Crucial factors are the efficiency, integrity and the indicated goodwill. The technology has to act conforming to the law, fair and transparent.

Perceived Lawfulness: Data should be processed in accordance with the law.

Fairness: The technology should represent the interests of all users fair, moderate unfair imbalances so that no user feels disadvantaged or exposed.

Transparency: Degree of transparency about how the technology uses and processes the data, i.e. the right to receive meaningful information about the logic when automated decision-making or profiling is used.
